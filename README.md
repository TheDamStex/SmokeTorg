# SmokeTorg

## 1. Опис проєкту
SmokeTorg — це настільна система для автоматизації роздрібної торгівлі у форматі **POS + склад + дисконтна система**.

Коротко:
- Призначення: облік продажів, приходу товарів, залишків і клієнтських карт.
- Цільова аудиторія: невеликі та середні магазини, що працюють з касою та складом.
- Основна мета: дати єдиний робочий інструмент для касира, менеджера та адміністратора.

---

## 2. Технології
- **.NET 8** (`net8.0-windows`)
- **WPF** + патерн **MVVM**
- **MySQL**
- **MySqlConnector**
- **Dependency Injection**: `Microsoft.Extensions.DependencyInjection`
- Архітектурний підхід: **Layered / Clean-like**
- Локальне зберігання налаштувань БД з шифруванням пароля через **DPAPI**

---

## 3. Архітектура
Проєкт поділено на логічні шари:

- `Domain` — сутності бізнес-домену та enum-и ролей/статусів.
- `Application` — інтерфейси та прикладні сервіси (auth, продажі, склад, користувачі тощо).
- `Infrastructure` — доступ до даних, SQL-скрипти, ініціалізація БД, збереження налаштувань.
- `Presentation` — WPF Views/ViewModels, вікна, діалоги, навігація.
- `Common` — базові класи MVVM, команди, логер, загальні утиліти.

Ключові принципи:
- Репозиторії використовуються через інтерфейси (`IProductRepository`, `IUserRepository`, ...).
- Весь склад об’єктів реєструється через DI-контейнер у `App.xaml.cs`.
- Підтримується перемикання типу зберігання через фабрику (`Json` / `Sql`), але у поточній конфігурації частина модулів працює через JSON-репозиторії, а користувачі — через MySQL.
- `DbInitializer` відповідає за перевірку підключення, створення БД та схеми.
- `SetupWizard` запускається на першому старті й проводить початкову конфігурацію.

## 4. Реалізований функціонал
### ✅ Реалізація (POS)
- Продаж товарів
- Робота зі штрихкодами
- Застосування дисконтної картки
- Розрахунок підсумку

### ✅ Прихід товару
- Додавання товарів
- Робота з постачальниками
- Формування закупівлі

### ✅ Склад
- Перегляд залишків
- Контроль мінімальних залишків

### ✅ Дисконтна система
- Список карт
- Створення карт
- Вибір картки у POS
- Типи знижок: бонусна та відсоткова

### ✅ Ролі користувачів
- `Admin`
- `Manager`
- `Seller` (у коді також підтримується синонім `Cashier`)

## 5. Перший запуск (Setup Wizard)
На першому запуску виконується майстер налаштування:

1. Налаштування підключення до MySQL (host/port/database/user/password).
2. Створення БД (якщо не існує).
3. Автоматичне створення таблиць та службової схеми.
4. Створення облікового запису адміністратора.
5. Збереження конфігурації для наступних запусків.

---

## 6. Структура бази даних
Основні таблиці:

- `Users` — користувачі системи, ролі, стан облікового запису, парольні хеші/сіль.
- `Products` — товари (SKU, штрихкод, ціни, мін. залишок, категорія).
- `Categories` — категорії товарів.
- `Suppliers` — постачальники.
- `Customers` — клієнти та дисконтні картки/бонусні параметри.
- `Sales` / `SaleItems` — документи продажу та позиції продажу.
- `Purchases` / `PurchaseItems` — документи закупівлі та позиції приходу.
- `SchemaInfo` — службова інформація про версію схеми БД.

Додатково використовуються таблиці `Settings`, `StockItems`, `StockMovements` для налаштувань і складського обліку.

---

## 7. Управління користувачами
- Створювати нові акаунти може користувач з роллю `Admin`.
- Доступ до налаштувань БД та системних параметрів має `Admin`.
- Перевірка ролей виконується у ViewModel-рівні (зокрема в головній моделі, керуванні користувачами та вікні налаштувань БД) та підтримується бізнес-сервісами.

---

## 8. Безпека
- Паролі зберігаються у вигляді хешів (**PBKDF2**) із сіллю.
- Пароль до БД у локальному конфігу шифрується через **DPAPI** (CurrentUser scope).
- Для SQL доступу використовуються параметризовані запити (через MySqlConnector), що знижує ризики SQL injection.

## 11. Як запустити проєкт
1. Встановити **.NET 8 SDK**.
2. Підготувати доступний екземпляр **MySQL**.
3. У корені рішення виконати:
   ```bash
   dotnet build SmokeTorg/SmokeTorg.csproj
   dotnet run --project SmokeTorg/SmokeTorg.csproj
   ```
4. На першому запуску пройти **Setup Wizard**.

> Примітка: застосунок є WPF (`net8.0-windows`), для повноцінного запуску потрібне середовище Windows.

## 12. Важливо для подальшої розробки
### Для AI / розробників
- Не використовувати кирилицю в назвах C#-класів, властивостей, методів і полів.
- Усі UI-тексти мають бути українською.
- Дотримуватись MVVM строго.
- Не розміщувати бізнес-логіку у `code-behind`.
- Усі нові модулі підключати через DI.
- Не ламати існуючі інтерфейси без окремого плану сумісності.
